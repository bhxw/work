**《人工智能》课程论文要求**

**选题方向（二选一）：**

**（一）复现任意人工智能算法**

学生可选择自构数据集或公开数据集，对任意一种人工智能算法进行完整复现。论文内容应包括算法原理介绍、模型搭建、实验平台、训练过程、实验结果展示与分析等部分。建议尽可能还原原始方法的实现细节，并通过实验结果对算法效果进行验证和讨论。

**（二）人工智能课程学习心得**

学生可围绕人工智能课程的学习过程撰写学习体会，论文结构可自行设计。内容可包括对人工智能基本概念、核心技术及典型应用的理解，学习过程中遇到的困难、解决方法与收获，以及对人工智能未来应用方向的思考等。建议结合课堂教学内容、实际案例或相关项目进行分析与总结，使论文内容更加具体、充实。

**论文要求：**

- 正文字数不少于 2000 字，不含参考文献。
- 正文字体为小四号，行距为 1.5 倍行距。
- 需注明课程论文来源，包括教材、论文、网页、博客、视频、开源项目、大语言模型生成内容等。
- 论文中出现的每一张图片均需注明来源。图片来源可以包括使用 PowerPoint、Visio 或 Adobe Illustrator 等工具自行绘制，来源于知乎、CSDN 等平台，来源于论文截图，或由大语言模型生成等（自行绘制可酌情加分）。
- 选择“（一）复现任意人工智能算法”方向的学生，请参考以下课程论文结构撰写，并可根据选题内容和实际完成情况适当调整、增删相关部分；选择“（二）人工智能课程学习心得”方向的学生，论文形式及结构不作特殊要求，可根据内容需要自行设计。

**课程论文结构**

1.  **课程论文来源说明**

本部分用于说明课程论文的资料来源、写作基础以及是否使用了相关辅助工具。学生应如实说明论文内容的主要来源，包括但不限于教材、课堂讲义、学术论文、网络资料、开源项目、知乎、CSDN、博客平台、视频课程、公开数据集，以及大语言模型等。

若论文参考了网络资料，应注明具体来源、作者或平台名称，并在参考文献或文末说明中列出相关链接。若论文内容参考了开源项目或算法代码，应注明项目名称、代码来源、项目链接以及本人在复现或修改过程中完成的工作。

若论文写作过程中使用了大语言模型辅助，应说明使用方式。例如，是否用于资料检索、概念解释、代码调试、论文润色、结构建议或图片生成等。学生应明确说明哪些部分由本人独立完成，哪些部分借助工具辅助完成。使用大语言模型并不直接影响成绩，但未说明使用情况将影响论文评价。

**示例写法：**

本文选题为“基于卷积神经网络的手写数字识别任务”。论文内容主要参考CSND博客资料：

”https://blog.csdn.net/qq_44807756/article/details/138193484”

实验代码由本人在参考该博客示例的基础上完成，并对模型结构、训练参数和结果分析进行了整理。论文写作过程中使用大语言模型辅助进行文字润色和概念解释，最终实验过程由本人完成。

2.  **摘要**

内容：简明扼要地介绍论文的研究背景、所选算法、主要实验结果及最终结论。

3.  **引言**

背景与动机：阐述人工智能在当前科技与应用中的重要性，说明选择该算法的原因与研究意义。

4.  **算法简介**

理论原理：详细说明算法的数学基础、核心思想。

5.  **实验设计与应用场景**

**（一）数据介绍**

- 数据集介绍：描述所选数据集的来源、规模及适用场景。
- 数据预处理：详细说明数据清洗、归一化等处理方法，并解释每步处理的目的。
- 数据可视化：利用图表（如直方图、散点图等）展示数据分布或预处理前后对比，增强直观性。

**（二）实验设置**

- 实验环境：介绍硬件配置、软件环境及运行平台。
- 参数设置：列出模型使用的主要参数及调优策略，说明参数选择依据。
- 对照实验：可设置对比实验（如与原算法或其他方法的比较）以验证算法性能。

6.  **实验结果与分析**

**（一）定量评估**

根据具体任务类型选择合适的性能评估指标。例如，分类任务可采用准确率、敏感性、特异性、召回率、精确率、F1 值、ROC 曲线、AUC 值等指标进行评价；回归任务可采用平均绝对误差（MAE）、均方误差（MSE）、均方根误差（RMSE）、决定系数（R²）等指标进行评价；目标检测任务可采用精确率、召回率、平均精度（AP）、平均精度均值（mAP）、交并比（IoU）等指标进行评价。论文中应提供完整的实验数据，并通过表格、曲线图、混淆矩阵或其他可视化方式展示模型性能，以便对实验结果进行客观分析和比较。

**（二）定性评估**

- 可视化分析：展示算法在不同样本或场景下的输出结果，进行结果可视化。
- 错误案例分析：挑选部分典型错误案例，探讨算法失误原因及可能的改进方案。

**（三）代码运行截图**

请提供代码运行过程中的截图，以展示程序的执行效果和结果。

**（四）讨论**

根据实验结果，讨论算法的优势、不足及实际应用中的适用性。

7.  **结论**

对课程论文进行总结。

8.  **参考文献**

列出所有参考资料，包括论文以及网络资料。

9.  **GitHub源代码链接（加分项，2-5分）**

- 代码上传：将完整代码上传至GitHub仓库，请勿将源码作为附录上传，也不要通过微信方式向教师提交代码。
- 运行指南：编写详细的README文件，说明代码依赖、环境配置、数据准备及运行步骤，确保他人能够复现实验结果。
- 项目结构：对项目文件结构、模块划分及代码注释进行说明，提升代码的可读性和维护性。
- 参考例子：https://github.com/guaguabujianle/E3RelaxH2

10.  **自构数据集（加分项，2-5分）**

学生可自行收集、整理数据集，并根据任务需要对数据进行标注。数据收集方式包括但不限于：使用相机、手机等设备自行拍摄获取数据；通过网络爬虫采集公开数据；或从 Kaggle 等公开数据平台下载并进一步整理和清洗。该项加分将综合考虑数据集的构建难度、整理质量、标注以及数据集的独特性。若论文所使用的数据集具有一定创新性、个性化特点，或在全班课程论文中使用频率较低，可酌情给予更高加分。

11.  **论文格式排版（加分项，2-5分）**

学生应注意课程论文的整体格式规范和版面美观。论文排版应结构清晰、层次分明，正文格式符合课程要求，图表、公式、参考文献等内容应统一规范。

- 该项加分将综合考虑以下方面：
- 论文标题、一级标题、二级标题等层级清楚，编号规范；
- 正文字体、字号、行距、段落间距等符合要求；
- 图表排版规范，图题、表题、编号和来源标注完整；
- 公式、变量符号和单位书写规范；
- 参考文献格式统一，文中引用与文末参考文献能够对应（可以使用EndNote或Zotero等工具）；
- 页面整体美观，内容排列整齐，无明显错别字、格式混乱或大段空白等问题。

**（1）公式规范**

公式放表格，表格外框隐去，当需要对公式进行解释时，公式后面加一个逗号，下一行无需首行缩进。例如：

|     |     |
| --- | --- |
|     | (2-1) |

当中， 代表 Frobenius 内积， 属于稳定项。这一指标可为后续时间门控与先验可信度分析给出直观说明。
